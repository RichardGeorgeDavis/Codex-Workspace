Codex Workspace

One workspace for many repos, without pretending they are one project.

Codex Workspace is a local-first structure for mixed stacks. It keeps repositories independently runnable, shares caches instead of installs, and pairs the workspace with a practical Workspace Hub for discovery, runtime control, and previews.

View Repository Read Docs Latest Release

Core model

  • Keep unrelated repositories independent.
  • Share caches, not installs.
  • Support mixed runtime models without flattening them.
  • Keep reverse-proxy or mapped-host tooling optional.

Workspace Hub

The vendored Workspace Hub app scans sibling repos, classifies them conservatively, exposes runtime state, and adds start, stop, open, and preview actions without forcing a monorepo.
